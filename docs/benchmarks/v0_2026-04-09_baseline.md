## 6. 已修复的 Bug (v0 开发期间)

| Bug | 根因 | 修复 |
|-----|------|------|
| Python.org 乱码 | reqwest 未启用 gzip/brotli | 加 feature flags |
| HN table 膨胀 14K→6K | layout table 被渲染成 md table | `is_data_table()` 启发式判断 |
| 导航栏重复 3 遍 | 未做行级去重 | `dedup_and_clean()` |
| MDN 内容全丢 0 chars | `[class*='banner']` 误杀布局容器 | 缩小 noise selector 范围 |
| lol_html `&#160;` 未解码 | 无 entity 解码 | `decode_entities()` |
| `<pre>` 代码缩进被压扁 | whitespace collapsing 在 pre 内 | `[[PRE_START/END]]` 状态标记 |
| 相对链接丢失 | 只匹配 `http` 开头 | base URL 拼接 |
| `<title>` 泄漏到 body | lol_html 未删 `<head>` | noise selector 加 `head` |

## 7. v1 待优化项

1. lol_html 大页面 O(n²) → 需要优化 `dedup_and_clean` 和 `decode_entities`
2. HN `| hide |` 噪声 (31 个) → 添加 site-specific 过滤规则
3. MDN 链接留存率 0% → 需要处理更多相对链接模式
4. T1 Chrome 路径未接 lol_html → 大页面仍用 scraper，性能差
5. PyO3 nested runtime panic → T1 模式在 PyO3 下不可用
